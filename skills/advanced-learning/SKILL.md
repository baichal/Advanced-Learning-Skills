# Advanced Learning Skills — 高级学习技能

## 首次激活检查

本技能每次被触发时，先检查初始化状态。

检查方式：尝试读取 `memory/knowledge/INDEX.md`。

- 文件存在且包含索引表头 → 初始化已完成，跳过，进入正常流程。
- 文件不存在或内容为空 → 首次激活，读取 `setup-checklist.md` 执行完整初始化。完成后再进入正常流程。

初始化过程静默执行，不向用户展示。如果初始化过程中遇到无法自行解决的问题（如没有文件写入权限），用自然语言告知用户需要协助，不暴露内部结构细节。

## 卸载

当用户明确说"卸载 advanced-learning""移除这个技能""删除学习技能"或类似表述时，
读取 `setup-checklist.md` 的卸载流程段落并执行。

## 核心理念

研究的目标不是让多个来源说一样的话，而是完整理解一个问题的信息全貌。
全貌包括：哪些部分有共识、哪些部分存在分歧、分歧背后各自的道理是什么、每种观点在什么条件下成立。
共识不自动等于正确，分歧不自动等于有一方错了。理解"为什么不同"比判定"谁对谁错"更重要。

## 触发条件

以下任一情况自动启用：
- 对答案置信度不足
- 问题涉及多个对立维度
- 信息可能过时
- 用户说"深入研究""帮我分析""详细调查"或类似表述

## 快通道

memory_search 命中相关知识条目后，执行条目验证：
1. 适用前提是否匹配当前问题
2. expires 字段是否已过期
3. 置信度是否为中以上

全部通过 → 直接使用条目内容回答。内部记录调用，更新 call_count。
部分通过 → 使用已知部分回答，不确定部分自然地询问用户是否需要展开。
未通过或未命中 → 进入慢通道。

以上检索和判断过程不向用户展示。

## 慢通道

### 第 1 轮：广度扫描
搜索获取至少 5 个不同来源。刻意寻求来源多样性——不同立场、不同背景、不同领域的视角。
逐一独立总结每个来源的核心观点、立场前提、信息质量。
不要在这一步就过滤或判断对错。先完整收集。
详细评估标准 → 读取 `research-methodology.md`

### 第 2 轮：全貌绘制
将所有来源放在一起，不是为了找出谁对谁错，而是绘制这个问题的信息全貌。对每个观点理解它成立的条件。

按以下维度整理：

| 维度 | 要回答的问题 |
|------|------------|
| 共识区域 | 哪些部分各方没有分歧？这些共识的基础是什么？ |
| 条件性结论 | 哪些结论只在特定条件下成立？各自的条件是什么？ |
| 视角差异 | 哪些来源从不同角度看同一个问题？每个角度揭示了什么？ |
| 根本性分歧 | 哪些来源在相同前提下得出矛盾结论？ |
| 未覆盖区域 | 有没有明显重要但所有来源都没讨论的方面？ |

### 第 3 轮：理解分歧
针对第 2 轮中发现的根本性分歧和未覆盖区域：
1. 搜索新来源，不是为了打破平局，而是为了理解分歧的根源
2. 对每个分歧点追问：
   - 双方各自的推理链是什么？哪一步开始分叉？
   - 分歧是来自不同的事实基础、不同的价值取向、还是不同的方法论？
   - 这个分歧在实践中意味着什么？对不同场景的用户有什么不同影响？
3. 对未覆盖区域：寻找专门讨论该方面的来源

如果第 3 轮揭示了新的维度，回到第 2 轮补充全貌。

归因分析框架 → 读取 `research-methodology.md`

### 停止条件

全部满足才停止：
- 每个已发现的观点都理解了它成立的条件和推理基础
- 各观点之间的关系已经清晰（互补、条件性差异、还是根本性分歧）
- 根本性分歧已经理解了分歧的原因（即使无法判定谁对）
- 已经主动寻找过不同立场的来源，不是只看了一个方向

存在任一则继续：
- 某个观点只知道结论不知道推理基础
- 有分歧但不理解为什么会分歧
- 所有来源都来自同一立场或同一类型
- 存在明显重要但未探索的维度

## 回答方式

根据理解到的信息全貌自然地组织回答。不展示内部流程。

**全貌清晰且有共识基础时：**
给出核心结论。如果存在条件性差异或少数不同视角，自然地带出来，说清楚各自成立的条件。不把某个视角当作"主流"去压制其他视角，而是让用户理解全貌后自己判断。

**多个视角各自成立时：**
呈现不同视角和各自的道理，不强行排出高下。如果从对话上下文能判断用户场景，说明在他的情况下哪个视角更相关以及为什么。如果判断不了，自然地问用户几个关键问题来帮助他匹配。

**存在根本性分歧时：**
如实呈现分歧，解释各方的推理和分歧的根源。不因为某个观点是"多数"就默认它正确。给出你的理解：在什么条件下各自的道理更站得住。如果你确实有判断倾向，说出来并解释为什么，同时承认另一方的合理性。

**信息不足时：**
直接说目前能找到的信息有限，说清楚哪些部分能确定、哪些不能、以及为什么不能。如果知道去哪里能找到更好的信息，自然地提一下。

## 静默沉淀

研究完成后在后台判断是否沉淀为长期知识：

**沉淀条件（全部满足）：** 通用性 + 经过验证 + 可复用
**不沉淀：** 高度个人化、一次性任务、强时效、纯上下文维护 → 仅写入当日 memory 笔记

沉淀时：先用 memory_search 去重 → 按结构写入 `memory/knowledge/{分类}/` → 更新 INDEX.md
完整结构定义和写入流程 → 读取 `knowledge-schema.md`

沉淀完整的信息全貌，不只沉淀"胜出"的结论。条件性结论记录各自的条件，分歧记录各方的推理和分歧原因。

整个沉淀过程不向用户展示任何信息。
